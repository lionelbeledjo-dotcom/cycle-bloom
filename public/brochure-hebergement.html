💡 Pourquoi cette stack ?

Supabase remplace Firebase + un serveur Node.js complet. Tu as auth, base de donnees, stockage fichiers, fonctions serverless et websockets — tout en un, avec un tier gratuit genereux.

  • Preview deploys par branche/PR
  • Analytics integres

Etapes pour deployer sur Vercel

1

Creer un compte Vercel

Va sur vercel.com et connecte-toi avec ton compte GitHub (lionelbeledjo-dotcom).

2

Importer ton repo

Clique "Add New Project" → selectionne ton repo (Lb-Group-Hotel ou cycle-bloom). Vercel detecte automatiquement Vite/React.

3

Configurer les variables d'environnement

Ajoute tes variables Supabase :

VITE_SUPABASE_URL=https://xxx.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=eyJ...
4

Deployer

Clique "Deploy". En 30 secondes, ton app est en ligne sur ton-projet.vercel.app. Chaque futur push GitHub = nouveau deploy automatique.

5

Ajouter ton domaine custom

Settings → Domains → Ajoute lb-group.com ou cyclebloom.ai. Vercel te donne les DNS a configurer chez ton registrar.

Comment configurer le DNS

1

Achete ton domaine

Sur Namecheap, Cloudflare ou Vercel. Tu recois un acces a la zone DNS.

2

Ajoute le domaine dans Vercel

Vercel te donne un enregistrement a ajouter :

Type: CNAME  |  Nom: @  |  Valeur: cname.vercel-dns.com
3

Ajoute l'enregistrement DNS chez ton registrar

Va dans la zone DNS de ton domaine et colle l'enregistrement. Propagation en 5-30 minutes.

4

HTTPS automatique

Vercel genere automatiquement un certificat SSL Let's Encrypt. Rien a faire, c'est instantane.

💡 Conseil : Cloudflare en proxy

Meme si tu heberges sur Vercel, utilise Cloudflare comme DNS (gratuit). Tu gagnes : cache mondial, protection DDoS, analytics DNS, et tu peux changer d'hebergeur sans modifier le domaine.

Publication App Store & Play Store

Transforme ton app web en app native avec Capacitor — sans reecrire le code

📱 Qu'est-ce que Capacitor ?

Capacitor (par Ionic) prend ta web app React et la "wrappe" dans un conteneur natif. L'app utilise un WebView optimise — tes utilisateurs ne voient aucune difference avec une app native. Tu gardes un seul codebase pour Web + iOS + Android.

🍎 Publication sur l'App Store (iOS)

1

Compte Apple Developer

Inscris-toi sur developer.apple.com. Cout : 99 $/an. Validation d'identite en 24-48h.

2

Installer Capacitor

Dans ton projet :

npm install @capacitor/core @capacitor/cli @capacitor/ios
npx cap init "CycleBloom AI" com.cyclebloom.app
npx cap add ios
3

Build & sync

Build ta web app puis synchro avec le projet natif :

npm run build && npx cap sync ios
4

Ouvrir dans Xcode

Necessite un Mac. Ouvre le projet iOS : npx cap open ios. Configure le signing, l'icone, le splash screen.

5

Soumettre a Apple

Archive → Upload sur App Store Connect → Remplis les metadonnees (description, screenshots, categorie "Health & Fitness") → Soumets pour review. Delai : 24-48h en moyenne.

🤖 Publication sur le Play Store (Android)

1

Compte Google Play Console

Inscris-toi sur play.google.com/console. Cout : 25 $ une seule fois (a vie).

2

Ajouter Android a Capacitor

npm install @capacitor/android
npx cap add android
npm run build && npx cap sync android
3

Generer l'APK/AAB signe

Ouvre Android Studio (npx cap open android), configure le keystore de signature, puis Build → Generate Signed Bundle (AAB).

4

Upload sur Play Console

Cree une fiche (titre, description, screenshots, categorie "Sante"), upload l'AAB, lance une release en "Production". Review : 1-3 jours.

⚠ Prerequis Apple

Tu as besoin d'un Mac (ou Mac en location cloud type MacStadium ~40$/mois) pour compiler l'app iOS. Pour Android, Windows suffit avec Android Studio.

Alternative : PWA (Progressive Web App)

Ton app installable sur mobile SANS passer par les stores — gratuit et instantane

App Native (Capacitor)

Via App Store / Play Store
  • Presence sur les stores (visibilite)
  • Push notifications natives
  • Acces capteurs (GPS, camera)
  • Credibilite aupres des users
  • 99$/an (Apple) + 25$ (Google)
  • Review obligatoire (24-72h)
  • Necessite un Mac pour iOS
  • Commission 15-30% sur achats in-app

PWA

Installable directement depuis le navigateur
  • Zero cout supplementaire
  • Zero review / validation
  • Mise a jour instantanee (pas de store)
  • Installable sur ecran d'accueil
  • Fonctionne offline (service worker)
  • Pas de commission sur les paiements
  • Pas de presence sur les stores
  • Push notifications limitees sur iOS

💡 Ma recommandation

Phase 1 (maintenant) — Lance en PWA. C'est gratuit, instantane, et tes utilisateurs peuvent installer l'app depuis leur navigateur. Ca te permet de valider ton produit sans depenser.

Phase 2 (quand tu as des revenus) — Ajoute les stores. Capacitor te permet de garder le meme code. L'investissement (124$/an + Mac) est justifie quand tu as des clients payants.

Activer la PWA sur ton projet

1

Installer vite-plugin-pwa

npm install vite-plugin-pwa -D
2

Configurer dans vite.config.ts

Ajoute le plugin VitePWA avec : nom, icones, couleur theme, strategie de cache.

3

Ajouter un manifest.json

Nom de l'app, icones 192x192 et 512x512, couleurs, orientation portrait.

4

Deployer

Push sur GitHub → Vercel deploy. Les utilisateurs voient "Ajouter a l'ecran d'accueil" automatiquement.

Paiements & Emails

Stripe pour encaisser, Resend pour communiquer — les deux s'integrent dans Supabase Edge Functions

💳

Stripe

Paiements en ligne — Abonnements, paiements uniques, factures. Supporte cartes, Apple Pay, Google Pay, Mobile Money (Afrique).

  • • Cout : 1.5% + 0.25€ par transaction (EU)
  • • Pas d'abonnement mensuel
  • • Dashboard : dashboard.stripe.com
  • • Test mode gratuit pour developper
Gratuit (commission uniquement)
📧

Resend

Emails transactionnels — Bienvenue, confirmation, reset password, factures, newsletters. Templates React Email.

  • • Free : 100 emails/jour, 3000/mois
  • • Pro : 20$/mois = 50 000 emails
  • • Domain custom pour delivrabilite
  • • Dashboard : resend.com/emails
100 emails/jour gratuit

Stripe : Etapes de mise en place

1

Creer un compte Stripe

Va sur stripe.com. Remplis tes infos entreprise/auto-entrepreneur. Verification d'identite en 24h.

2

Creer tes produits & prix

Dans le dashboard Stripe : Products → Cree "Premium Mensuel" (9.99€/mois recurring), "Premium Annuel" (79.99€/an recurring).

3

Integrer via Supabase Edge Function

Cree une Edge Function qui genere un "Checkout Session" Stripe. L'utilisateur est redirige vers la page de paiement Stripe securisee.

4

Webhook pour confirmer

Stripe envoie un webhook quand le paiement est confirme → ta Edge Function met a jour le role de l'user dans Supabase (free → premium).

💰 Astuce Mobile Money (Afrique)

Stripe supporte Mobile Money (Orange Money, MTN MoMo) dans certains pays africains. Alternative : integre PayDunya ou CinetPay en parallele pour couvrir le Cameroun et l'Afrique de l'Ouest avec un maximum de methodes de paiement locales.
